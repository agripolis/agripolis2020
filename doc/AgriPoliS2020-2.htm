Issues of AgriPoliS-2

(Changxing Dong, July 2018)

 

AgriPoliS2020: Reorganization of input

 

Refactoring steps to the start_version of AgriPoliS2020

The start_version of AgriPoliS2020 uses  the  Altmark as the reference region with new formats for the matrix and policy files in addition to changes to other input files. Starting from this version, AgriPoliS is further refactored with the following operations:

1.      Code cleaning.

Subtasks include: removing unused global INTEGER_SWITCH; removing some unused functions and variables; removing obsolete documentation; translating some documentation lines into English; removing unused files.

2.      Command line options.

Command line options for RUNS and TEILER are introduced. This is mainly for debugging’s convenience, not for normal use. Normal users can ignore these command line options.

3.      Reorganization of inputs/outputs.

This operation aims at improving the usability of AgriPoliS, especially the readability of the inputs. The improvement is carried out by separating financial and labour related items from normal products or investments.

 

Details of the input reorganization

The last refactoring operation affects the usage of AgriPoliS directly, especially for users of earlier versions. Here are the details.

1.      Remove PRODUCTGROUPS etc. from globals.txt.  PRODUCTGROUPS is directly related to the column “group” in market.txt and should be in accordance with each other.  This is guaranteed with this refactoring step, since PRODUCTGROUPS is calculated hereafter in the program.

2.      Remove invest_groups from globals.txt. The value of it is assigned hereafter also through the program.

3.      Remove OFFFARMLABTYPE etc. from globals.txt. These are constants which correlate to the column “type” in market.txt or investments.txt and are now considered directly through the program.

4.      Reorganize the entries in globals.txt and options.txt according to their meanings; Introduce groupings for management, production, finance, land, labour, etc. in globas.txt.

5.      ST_BORROW_CAP and EC_INTEREST are moved from market.txt to globals.txt, under finance group.

6.      V_HIRED_LABOUR and V_OFF_FARM_LAB are moved from market.txt to globals.txt; HIREDLAB and OFFFARMLAB are moved from investments.txt to globals.txt. All under labour group.  With these changes the "translation"-lines at the beginning of investments.txt are not necessary any more.

7.      A scenario file is introduced to the input, which has the default file name scenario.txt and the following format:

Scenario:   renting_strategy

Description:  to investigate the combinational effects of rent_adjust_factor and generation_change

Policy_file:  policy_settings.txt

 

[globals]

rent_adjust_factor           0.5

generation_change          15

Where “Scenario:” “Description:” “Policy_file:” and “[globals]” are part of the format. The globals listed under [globals] override the corresponding values in globals.txt. The scenario name will be in the output files.

 

Effects of the input reorganization

The Effects of the reorganization of input files can be summarized as:

1.      With groupings, the file globals.txt is more structured. This greatly improves the readability of this file.

2.      Labour related variables are grouped in globals.txt, not distributed over market.txt and investments.txt.

3.      Financial products are moved from market.txt to globals.txt, under finance group.

4.      Both market.txt and investments.txt are purer without labour and finance related items.

5.      Scenario file is introduced with scenario.txt as the default file name.  It can be called otherwise and there can be more than one scenario files in the same input folder. In this case, the scenario file should be explicitly given in the command line after the input folder name.
