Issues of AgriPoliS-5

(Changxing Dong, November 2018)

 

Extension with Multiple Random Number Generators

 

Random number generators play an important role in simulations with stochastic processes, such as those carried out by AgriPoliS.  There are several processes in AgriPoliS which need a random number generator.  These processes are independent to each other and it should be possible to investigate their influences independently. However, up until now, this has not been possible since there is only one random number generator in the program. This extension aims to solve this issue.

In this extension, eight new random number generators are implemented to make it possible to investigate the influences of their corresponding processes individually. The names of the random number generators and the corresponding processes are:

1.      mgmtCoeff                        assigning management coefficient to farm agents

2.      farmAge                             assigning initial farm age to farm agents

3.      closeFarm                          deciding whether a farm will close or not (effective only when NO_SUCCESSOR_BY_RANDOM is true)

4.      investAge                           assigning initial ages to the investments of each farm agent

5.      contractLengthInit           setting contract length of initial land input

6.      contractLength                setting contract length for newly rented land

7.      freePlot_initLand              choosing plot by landscape initialization

8.      freePlot_rentPlot             choosing plot to be rented (effective only when OLD_LAND_RENTING_PROCESS is true)

For every random number generator, five different algorithms from C++ standard library are provided:  minstd_rand0, minstd_rand, knuth_b, mt19937, mt19937_64. The first two implement the linear congruential algorithm, the last two implement the Mersenne Twister algorithm, and knuth_b is an engine adaptor to minstd_rand0.  

In addition to the possibility to choose the engine algorithm, the user can also set the seed to the random number generators instead of using the default ones.

The settings of the random number generators should be put in the file scenario.txt with the following format:

               # set seed of   random number generator (RNG)

               Seed_<RNG>      <number>           

               # set random number generator algorithm

               Rand_gen_<RNG>            <algorithm>

 

Example lines:

               Seed_mgmtCoeff             137

               Rand_gen_mgmtCoeff    mt19937

               Seed_freePlot_initLand                  11

               Rand_gen_freePlot_initLand        mt19937_64

               Rand_gen_contractLength            knuth_b

 

Without settings for a random number generator in scenario.txt, the algorithm will be minstd_rand0, and the seed will be the default value of the corresponding algorithm. The default seeds of these engine algorithms are:

               minstd_rand0     1

               minstd_rand       1

               mt19937              5489

               mt19937_64       5489

               knuth_b               1
