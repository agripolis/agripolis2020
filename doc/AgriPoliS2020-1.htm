Issues of AgriPoliS

(Changxing Dong, April 2018)

 

Start version for AgriPoliS2020

 

For the DFG project AgriPoliS2020, Altmark is used as the reference region. For this region, simulation results with old versions of AgriPoliS are available. With new versions of AgriPoliS, translations.txt is removed, market.txt and investments.txt are adjusted, new formats for matrix_new.txt and policy_settings.txt are used. To ensure the correctness of both the program and the input data, comparisons of the simulation results with new versions of program and input to those with old versions are carried out iteratively. To ease the description, the versions of the program and the input will be referred as AgriPoliS-old, input-old and AgriPoliS-new, input-new. The results are summarized as the following points.

1.      Regional_milk_quota

This is a global of input-old in globals.txt. Due to its dependence on Teiler (also in globals.txt), the program is changed so that it will not be read in from input, but calculated in the program. It is the sum of this quantity of the individual farms.

2.      INCREASEPRICE

This is a global of input-old in globals.txt. It disappears in input-new. Instead, price changes for V_HIRED_LABOUR_H and V_OFF_FARM_LAB must be provided for input-new in policy_settings.txt.

3.      Calculation of expected prices (in RegMarket.cpp)

The calculation of expected prices in AgriPoliS-old is wrong. This is fixed in AgriPoliS-new.

4.      New entries in matrixLinks.txt

In input-old, the values for the cells

(liquidity, financing_rule)x (BIOGAS150KW_N, BIOGAS450KW_N, BIOGAS800KW_N)

are fixed values.  In input-new they are dynamic values through new entries in matrixLinks.txt.

5.      Changed Data  

Two values in the matrix are corrected. The new values come from Florian Heinrich.  

(energy_requirements_livestock, MAIZE_SILAGE) and (requirements_maize_silage, MAIZE_SILAGE):

from      -88.179                to           -108.10

(requirements_rye, RYE_BIOGAS):

from      -4.845                  to           -48.45

6.      Tranch_i_deg

With AgriPoliS-new the tranch_i_deg (with i=1,2,3,4,5) in policy_settings.txt take effect also for the first simulation period. For comparison to AgriPoliS-old, the tranch part in policy_settings.txt should be removed (e.g. commented out).

7.      AGRI_SERVICE

In translations.txt of input-old, the AGRI_SERVICES is wrongly written as AGRI_SERVICE. This means AGRI_SERVICES in market.txt is not a special product as intended. For comparison, the stdName of AGRI_SERVICES in market.txt for input-new should be “-“.

8.      Labour substitutions for investments

The labour substitutions for investments are read in from investments.txt as integers for AgriPoliS-new. Whereas theses values are read in from the matrix of input-old as float values. For comparison, these values in input-old should be rounded to integers if they have fractional part (only three of them do).
